Run your own copy

The bot is open source and runs anywhere Python does. The easiest route is Render (free tier works), but Docker and a plain VPS are covered too.

On this page
1. Get credentials 2. Deploy on Render 3. Docker 4. Manual VPS 5. Environment variables 6. Session string 7. Firebase (optional) 8. Troubleshooting

1. Get your credentials

You need three things before anything else:

Telegram API ID & Hash

  • Go to my.telegram.org and log in with your phone number
  • Open API development tools and create a new application
  • Copy the api_id and api_hash

Bot token

  • Open @BotFather in Telegram and send /newbot
  • Follow the prompts, then copy the token (123456:ABC-DEF…)

Your admin ID

  • Message @userinfobot — it replies with your numeric user ID
🔒

Never commit tokens or API hashes to a public repo. Use environment variables, always.

2. Deploy on Render (recommended)

The repo ships with a ready-made render.yaml and start.sh, so deployment is mostly clicking:

  • 1. Create an account at render.com and fork/clone the repository to your GitHub
  • 2. Dashboard → New + → Background Worker → connect your repo
  • 3. Runtime: Python 3 (3.10+). Plan: Standard is recommended — video processing is hungry
  • 4. Build command:
build command
apt-get update && apt-get install -y --no-install-recommends ffmpeg mediainfo fonts-noto-core fonts-noto-color-emoji fontconfig libass9 && rm -rf /var/lib/apt/lists/* && pip install -r requirements.txt
  • 5. Start command: bash start.sh
  • 6. Add the environment variables from the table below (API_ID, API_HASH, BOT_TOKEN, BOT_NAME, ADMIN_IDS at minimum)
  • 7. Hit Create Background Worker and watch the logs — first build takes a few minutes
💡

ffmpeg, mediainfo and the Noto fonts are installed by the build command — the bot needs them for splitting, subtitle embedding and thumbnails.

3. Docker

If you prefer containers, fill in the required section only at the top of CONFIG/_config.py (bot name, admin IDs, API credentials, token) and build:

bash
docker compose up -d --build

The Docker setup also serves cookie files to the bot through the configuration-webserver container — that's what the default COOKIE_URL values point at.

4. Manual VPS setup

bash — Ubuntu / Debian
# system deps
sudo apt update && sudo apt install -y python3.10 python3-pip ffmpeg mediainfo \
  fonts-noto-core fonts-noto-color-emoji fontconfig libass9

# code
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO
pip install -r requirements.txt

# edit CONFIG/_config.py with your credentials, then:
python magic.py

For 24/7 uptime, wrap it in systemd or run inside tmux/screen. A small 1 vCPU / 2 GB box handles light usage; video+subtitles work better with 4 GB+.

6. Session string (optional)

Bots can't read a channel's admin log. If you want the channel-guard features, generate a user session string:

bash
python generate_session_string.py

Paste the output into CHANNEL_GUARD_SESSION_STRING. Treat it like a password — it is one.

7. Firebase (optional)

For multi-instance setups or a managed database, flip USE_FIREBASE=true and fill in the Firebase block in the config (email, password, and the web app config object: apiKey, authDomain, projectId, databaseURL, …). With Firebase off, the bot keeps everything in local JSON — simpler and perfectly fine for a single instance.

8. Troubleshooting

SymptomFix
Bot doesn't startRe-check API_ID, API_HASH and BOT_TOKEN. Read the service logs for the exact error.
Downloads failMake sure ffmpeg is installed (it's in the build command). Check disk space. For YouTube, refresh cookies.
YouTube asks to "verify you're human"Update your cookies, or enable the PO-token provider (YOUTUBE_POT_ENABLED) with a bgutil instance.
Out of memoryUpgrade the plan, or lower MAX_FILE_SIZE_GB in the limits config.
Bot ignores messages in a groupAdd the group ID to ALLOWED_GROUP_IDS — or leave it empty to allow all groups.
